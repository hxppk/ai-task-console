# AI 建券任务管理系统 — 产品设计文档

## 概述

AI 驱动的建券任务管理系统，分为两个独立产品面：

- **AI 任务后台**：面向产品、技术研发和算法，完整的任务管控能力
- **业务 AI 建券列表**：面向运营和业务方，简化视图 + 确认操作

两者共享同一套底层数据，独立页面设计。

## 技术选型

- React 19 + Vite + React Router
- Ant Design (antd) 组件库
- 页面框架复用策略实验室顶部导航和左侧菜单样式
- 纯前端可交互原型，使用 mock 数据，不对接后端

## 任务来源

两种触发方式，后续流程一致：

1. **OA 审批**：监听 OA 活动申请，审批通过后自动获取 Excel 并创建任务
2. **手动上传**：运营在业务 AI 建券列表页手动上传 Excel 创建任务

每次上传/获取包含两个 Excel 文件：建券字段 + 门店清单。

## 数据模型

### 层级关系

- **任务**（父级）：对应一次 OA 审批或一次 Excel 上传，包含多张券
- **券**（子级）：任务下的每一张优惠券，串行由 AI Agent 逐张创建

### 任务状态

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| 解析中 | Excel 正在解析拆券 | 收到 Excel |
| 执行中 | 券正在串行创建草稿 | 解析完成，开始建券 |
| 待确认 | 所有券草稿创建成功 | 全部券创建成功，无异常 |
| 已完成 | 所有券都已确认提交 | 全部券被运营确认 |
| 部分异常 | 部分券创建失败 | 有券失败且有券成功 |
| 异常 | 全部券创建失败 | 所有券都失败 |

状态流转：

```
解析中 → 执行中 → 待确认 → 已完成
              ↓
          部分异常 ──(重试)──→ 执行中
              ↓
            异常 ──(重试)──→ 执行中
```

关键规则：
- 任务必须所有券都创建成功，才能进入"待确认"
- 有任何券失败则停在异常状态，需人工点击重试
- 重试后回到"执行中"，只重跑失败的券

### 券状态

**AI 任务后台视角**：

| 状态 | 含义 |
|------|------|
| 排队中 | 等待前面的券处理完（串行） |
| 创建中 | AI Agent 正在执行 |
| 已完成 | 草稿创建成功 |
| 失败 | 创建过程出错，可重试 |

**业务 AI 建券列表视角**（简化）：

| 状态 | 含义 |
|------|------|
| 创建中 | AI 在处理（合并了排队中+创建中） |
| 待确认 | 草稿已生成，等运营确认 |
| 已确认 | 运营已确认提交 |

### 券执行链路

单张券的 AI 执行步骤（在 AI 任务后台的券详情面板中展示）：

1. 解析参数 — 从 Excel 提取字段
2. 填写表单 — Skill 执行表单回填
3. 校验 — 验证填写结果
4. 生成草稿 — 创建草稿完成
5. 确认提交 — 运营在草稿页确认（被动等待）
6. 结果回传 — 业务侧回传提交结果（可选，先加上）

## 页面设计

### AI 任务后台（3 个页面）

#### 页面 1：任务列表

**导航**：
- 顶部：策略实验室导航栏，高亮「AI 任务」
- 左侧：任务管理（当前页）

**统计卡片**（4 张）：
- 总任务数 / 执行中 / 部分异常 / 异常

**筛选栏**：
- 搜索框：支持任务编号、任务名称
- 状态筛选：全部 / 解析中 / 执行中 / 待确认 / 已完成 / 部分异常 / 异常
- 来源筛选：全部 / OA审批 / 手动上传

**任务表格**：

| 列 | 说明 |
|----|------|
| 任务编号 | 格式 TSK-YYYYMMDD-NNN |
| 任务名称 | 关联活动名称 |
| 来源 | OA审批 / 手动上传 |
| OA单号 | OA 来源显示，手动上传显示"—" |
| 券数量 | 该任务下总券数 |
| 完成进度 | 进度条 + 数字（如 3/5） |
| 状态 | 任务状态标签 |
| 发起人 | OA 审批人 / 上传者 |
| 创建时间 | |
| 操作 | 查看详情；异常状态额外显示"重试" |

**分页**：10/20/50 条切换

#### 页面 2：任务详情

**面包屑**：任务管理 / TSK-20260308-001

**任务头部**：任务编号 + 状态标签 + 任务名称

**来源信息卡片**（横排）：
- OA单号 / 来源 / 关联活动 / 发起人 / 创建时间

**任务进度步骤条**（水平）：
- 解析 → 执行 → 待确认 → 已完成
- 异常状态在对应节点上标红提示
- 每步显示耗时

**券清单表格**：

| 列 | 说明 |
|----|------|
| 券名称 | |
| 券类型 | 满减 / 折扣 / 立减 |
| 面额 | |
| 券状态 | 排队中 / 创建中 / 已完成 / 失败 |
| 耗时 | 创建完成后显示 |
| 操作 | 查看详情；失败时显示"重试" |

点击「查看详情」打开右侧券详情面板。

#### 页面 3：券详情面板（右侧抽屉）

从任务详情的券清单点击「查看详情」滑出。

**券基本信息**：
- 券名称 / 券类型 / 面额 / 适用范围 / 有效期 / 库存

**券执行链路**（垂直 timeline）：
- 解析参数 → 填写表单 → 校验 → 生成草稿 → 确认提交 → 结果回传
- 每步显示状态（完成/进行中/待执行/失败）和耗时

**草稿链接**：点击可跳转草稿页

**异常记录**：失败时显示错误信息和失败步骤

**底部操作**：
- 失败状态：「重试」按钮
- 待确认状态：「去确认草稿」按钮

**券清单表格**：

| 列 | 说明 |
|----|------|
| 券名称 | |

| 面额 | |
