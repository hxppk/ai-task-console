# AI 任务后台多 Skill 模型 — 设计文档 v2

## 概述

AI 任务后台从"建券专用"升级为**通用 AI 任务调度平台**。一个父任务（活动）下包含多个 Skill 执行，建券只是其中之一。业务建券列表只展示建券分支的具象化信息。

## 数据模型

### 三层结构

```
父任务（活动）                    ← 第一层：一个 OA = 一个活动
├── Skill: 建活动（信息+玩法）     ← 第二层：子任务（Skill 执行）
├── Skill: 创建人群
├── Skill: 建活动落地页
└── Skill: 建券
    ├── 券1: 满30减5              ← 第三层：Skill 产出条目（仅建券有多个）
    ├── 券2: 8折券
    └── 券3: 新客立减
```

### 规则

- 一个 OA = 一个父任务 = 一个活动（不考虑一个 OA 多活动）
- 前三个 Skill（建活动/创建人群/落地页）各产出 1 个条目
- 建券 Skill 产出 N 张券
- 各 Skill 之间无依赖，可并行执行

### 父任务

| 字段 | 说明 |
|------|------|
| id | 任务编号 TSK-YYYYMMDD-NNN |
| name | 活动名称 |
| source | OA审批 / 手动上传 |
| oaNumber | OA 单号（手动上传为空） |
| activity | 关联活动名称 |
| subTaskCount | 子任务（Skill）数量 |
| completedCount | 已完成的子任务数 |
| adminStatus | 管理状态（待创建/解析中/创建中/待确认/已完成/部分异常/创建异常） |
| bizStatus | 业务状态（创建中/待确认/已完成） |
| confirmedCount | 已确认的券数量（业务侧用） |
| totalCouponCount | 券总数（业务侧用） |
| creator | 发起人 |
| createdAt | 创建时间 |

### 子任务（Skill 执行）

| 字段 | 说明 |
|------|------|
| id | 子任务编号 SUB-NNN |
| name | 子任务名称 |
| skillType | Skill 类型：建活动 / 创建人群 / 建活动落地页 / 建券 |
| status | 状态：排队中 / 创建中 / 已完成 / 失败 |
| itemCount | 产出条目数（建券=N，其他=1） |
| completedItemCount | 已完成条目数 |
| duration | 耗时 |
| execSteps | 执行链路 |
| errors | 异常记录 |
| items | 产出条目列表（详见下方） |

### 产出条目（Skill 产出的具体内容）

通用字段 + ext 扩展字段，ext 内容由 skillType 决定：

**建券 ext**：type(券类型), amount(面额), scope(适用范围), validRange(有效期), stock(库存), draftId(草稿链接), bizStatus(业务状态)

**建活动 ext**：activityType(活动类型), startTime, endTime, budget(预算)

**创建人群 ext**：crowdSize(人群规模), tags(标签), crowdId

**建活动落地页 ext**：pageUrl(页面链接), templateName(模板名称)

#### 任务详情

- 来源信息卡片
- 任务进度步骤条
- **子任务清单**（核心）：

| 列 | 说明 |
|----|------|
| 编号 | SUB-xxx |
| 名称 | |
| Skill 类型 | 建活动 / 创建人群 / 建活动落地页 / 建券（Tag 样式） |
| 产出数 | 建券显示 N，其他显示 1 |
| 状态 | |
| 耗时 | |
| 操作 | 查看详情（开抽屉） |

#### 子任务详情抽屉

抽屉标题：子任务名称 + Skill 类型 Tag

**通用区域**：
- 执行链路 timeline
- 异常记录
- 操作按钮（重试/去确认草稿）

**扩展信息区域**（根据 Skill 类型渲染）：
- 建活动：活动类型、起止时间、预算
- 创建人群：人群规模、标签、人群ID
- 建活动落地页：页面链接、模板名称
- 建券：产出券列表（小表格：券名称/类型/面额/状态），每张券的状态独立
